# 技能：异常考勤说明

## 1. 业务流程

单步调用：`oa-abnormal-attendance-handling`

调用参数：
```json
{
  "abnormalAttendanceParam": {
    "staffDomain": "{staff_domain}",
    "reason": "{reason}",
    "supplementaryPunchTime": "{punch_time}",
    "type": "{punch_type}"
  }
}
```

## 2. 槽位清单

| 参数 | 说明 | 必填 | 类型 | 处理方式 |
|------|------|:----:|------|----------|
| punch_type | 补卡类型 | ✅ | 枚举 | 签到/签退 |
| punch_time | 补卡时间 | ✅ | string | YYYY-MM-DD HH:mm:ss |
| reason | 异常原因 | ✅ | 枚举 | 见下表 |

### reason 可选值

| 值 | 说明 |
|-----|------|
| `管理者的工作安排` | 因管理者安排导致的考勤异常 |
| `卡机故障或系统问题（公司已发通知）` | 系统故障导致的异常 |
| `极端天气或突发情况（公司已发通知）` | 天气或突发情况 |
| `新入职无卡/无打卡权限` | 新员工未配置打卡权限 |

## 3. 状态机

### [GATHERING] 信息收集态
**任务**：逐个收集缺失参数

**追问示例**：
- 缺少类型：`请问是签到还是签退？`
- 缺少时间：`请问补卡时间是？`
- 缺少原因：`请问异常原因是什么？`

### [PENDING] 待确认态
**确认卡片模板**：
```
📋 异常考勤说明确认

• 补卡类型：{punch_type}
• 补卡时间：{punch_time}
• 异常原因：{reason}

确认提交请回复"确认"，取消请回复"取消"
```

### [EXECUTING] 执行态
**成功响应**：
```
✅ 考勤异常说明已接受

相关记录已更新，请关注考勤系统变化。
```

## 4. 核心约束

### 4.1 适用范围（严格限定）
**✅ 可用场景**：
- 管理者的工作安排
- 卡机故障或系统问题（公司已发通知）
- 极端天气或突发情况（公司已发通知）
- 新入职无卡/无打卡权限

**❌ 不可用场景**：
- 忘带卡
- 未打上卡
- 迟到/早退
- 公出

### 4.2 证明要求
- 如因"卡机故障或系统问题"、"极端天气或突发情况"申请
- 需确保**公司已发布相关通知**作为依据

## 5. 示例对话

```
用户：我要提交异常考勤说明
Agent：好的，请问是签到还是签退？
用户：签到
Agent：请问补卡时间是？
用户：今天上午9点
Agent：（调用 time_oracle）请问异常原因是什么？
  - 管理者的工作安排
  - 卡机故障或系统问题（公司已发通知）
  - 极端天气或突发情况（公司已发通知）
  - 新入职无卡/无打卡权限
用户：卡机故障
Agent：
📋 异常考勤说明确认

• 补卡类型：签到
• 补卡时间：2026-02-26 09:00:00
• 异常原因：卡机故障或系统问题（公司已发通知）

确认提交请回复"确认"，取消请回复"取消"

用户：确认
Agent：✅ 考勤异常说明已接受，相关记录已更新。
```

## 6. 错误处理

| 错误场景 | 处理方式 |
|----------|----------|
| 原因不符合范围 | 提示"该原因不在适用范围内" |
| 缺少公司通知 | 提示"需确保公司已发布相关通知" |
| 网络错误 | 提示稍后重试 |
