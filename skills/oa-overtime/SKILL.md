# 技能：加班申请

## 1. 业务流程

单步调用：`oa-overtime-application`

调用参数：
```json
{
  "overtimeApplicationParam": {
    "overtimeDate": "{overtime_date}",
    "overtimeHours": {overtime_hours},
    "reason": "{reason}",
    "staffDomain": "{staff_domain}"
  }
}
```

## 2. 槽位清单

| 参数 | 说明 | 必填 | 类型 | 处理方式 |
|------|------|:----:|------|----------|
| overtime_date | 加班日期 | ✅ | string | 格式：YYYY-MM-DD |
| overtime_hours | 加班时长 | ✅ | number | 单位：小时，最小0.5小时起算 |
| reason | 加班原因 | ✅ | string | 需具体明确 |

## 3. 状态机

### [GATHERING] 信息收集态
**任务**：逐个收集缺失参数

**追问示例**：
- 缺少日期：`请问加班日期是哪天？`
- 缺少时长：`请问加班多长时间？（小时）`
- 缺少原因：`请问加班原因是什么？`

**退出条件**：所有必填参数收集完成

### [PENDING] 待确认态
**任务**：展示确认卡片

**确认卡片模板**：
```
📋 加班申请确认

• 加班日期：{overtime_date}
• 加班时长：{overtime_hours} 小时
• 加班原因：{reason}

确认提交请回复"确认"，取消请回复"取消"
```

**退出条件**：
- 用户回复"确认" → 进入 EXECUTING
- 用户回复"取消" → 终止流程

### [EXECUTING] 执行态
**任务**：调用 MCP 工具提交申请

**成功响应**：
```
✅ 加班申请提交成功

您的加班申请已进入审批流程，请注意查收审批通知。
```

**失败响应**：
```
❌ 加班申请提交失败

原因：{error_message}
```

## 4. 核心约束

### 4.1 真实性要求
- 仅限**真实工作需要**申请加班
- 禁止虚假申请

### 4.2 时长规范
- 加班时长必须为**正数**
- 最小单位：**0.5 小时**

### 4.3 原因明确性
- 加班原因需**具体明确**
- ❌ 不可填写："无"、"随便"、"其他"等模糊内容
- ✅ 应清晰阐述工作任务来源（如项目交付节点临近、处理突发业务等）

### 4.4 补偿依据
- 加班补偿将根据国家相关法律法规及公司政策执行
- 工作日加班、休息日加班及法定节假日加班可能适用不同的补偿标准

## 5. 示例对话

**示例1：完整流程**
```
用户：我要申请加班
Agent：好的，请问加班日期是哪天？
用户：明天
Agent：（调用 time_oracle）请问加班多长时间？
用户：3小时
Agent：请问加班原因是什么？
用户：项目上线紧急修复
Agent：
📋 加班申请确认

• 加班日期：2026-02-27
• 加班时长：3 小时
• 加班原因：项目上线紧急修复

确认提交请回复"确认"，取消请回复"取消"
用户：确认
Agent：✅ 加班申请提交成功，已进入审批流程。
```

**示例2：原因不明确**
```
用户：我要申请明天的加班，4小时
Agent：请问加班原因是什么？
用户：其他
Agent：加班原因需要更具体一些，请详细说明工作内容。
用户：系统故障排查
Agent：📋 加班申请确认...
```

## 6. 错误处理

| 错误场景 | 处理方式 |
|----------|----------|
| 时长 ≤ 0 | 提示"加班时长必须为正数" |
| 原因模糊 | 提示"请提供具体加班原因" |
| 日期已过 | 提示"不能申请过去的加班" |
| 网络错误 | 提示稍后重试 |
