# 技能：公出（本地）申请

## 1. 业务流程

单步调用：`oa-public-application-form`

调用参数：
```json
{
  "form": {
    "staffDomain": "{staff_domain}",
    "startTime": "{start_time}",
    "endTime": "{end_time}",
    "leaveReason": "{reason}"
  }
}
```

## 2. 槽位清单

| 参数 | 说明 | 必填 | 类型 | 处理方式 |
|------|------|:----:|------|----------|
| start_time | 开始时间 | ✅ | string | YYYY-MM-DD HH:mm |
| end_time | 结束时间 | ✅ | string | YYYY-MM-DD HH:mm |
| reason | 公出事由 | ✅ | string | 用户输入 |

## 3. 状态机

### [GATHERING] 信息收集态
**任务**：逐个收集缺失参数

**追问示例**：
- 缺少时间：`请问公出时间是？`
- 缺少事由：`请问公出原因是什么？`

### [PENDING] 待确认态
**确认卡片模板**：
```
📋 公出申请确认

• 公出时间：{start_time} 至 {end_time}
• 公出事由：{reason}

确认提交请回复"确认"，取消请回复"取消"
```

### [EXECUTING] 执行态
**成功响应**：
```
✅ 公出申请提交成功

申请单号：{application_id}
公出期间视为正常出勤，已进入审批流程。
```

## 4. 核心约束

### 4.1 考勤规则
- 公出期间视为**正常出勤**，不计入请假或缺勤

### 4.2 时间校验
- 开始时间必须早于结束时间
- 公出时段不能与已有考勤记录冲突

### 4.3 时长要求
- 单次公出时长超过 **8 小时**需附加详细的工作内容说明

### 4.4 非工作时间
- 公出期间如遇非工作时间，需在事由中特别说明

## 5. 示例对话

```
用户：我要公出
Agent：好的，请问公出时间是？
用户：今天下午2点到5点
Agent：（调用 time_oracle）请问公出原因是什么？
用户：去客户现场开会
Agent：
📋 公出申请确认

• 公出时间：2026-02-26 14:00 至 2026-02-26 17:00
• 公出事由：去客户现场开会

确认提交请回复"确认"，取消请回复"取消"

用户：确认
Agent：✅ 公出申请提交成功，已进入审批流程。
```

## 6. 错误处理

| 错误场景 | 处理方式 |
|----------|----------|
| 时间冲突 | 提示"该时间段已存在考勤记录" |
| 时长过长（>8h） | 提示"需附加详细工作说明" |
| 网络错误 | 提示稍后重试 |
