# 技能：出差考勤申请

## 1. 业务流程

单步调用：`oa-business-trip-attendance`

调用参数：
```json
{
  "businessTripAttendanceVO": {
    "staffDomain": "{staff_domain}",
    "reason": "{reason}",
    "tripType": "{trip_type}",
    "startAddress": "{start_address}",
    "targetAddress": "{target_address}",
    "startTime": "{start_time}",
    "endTime": "{end_time}"
  }
}
```

## 2. 槽位清单

| 参数 | 说明 | 必填 | 默认值 | 处理方式 |
|------|------|:----:|--------|----------|
| start_time | 开始日期 | ✅ | - | YYYY-MM-DD HH:mm |
| end_time | 结束日期 | ✅ | - | YYYY-MM-DD HH:mm |
| reason | 出差事由 | ✅ | - | 用户输入 |
| trip_type | 出差类型 | ✅ | - | 国内/国际 |
| start_address | 始发地 | ❌ | - | 用户输入 |
| target_address | 目的地 | ❌ | - | 用户输入 |

## 3. 状态机

### [GATHERING] 信息收集态
**任务**：逐个收集缺失参数

**追问示例**：
- 缺少时间：`请问出差时间是？`
- 缺少事由：`请问出差原因是什么？`
- 缺少类型：`请问是国内出差还是国际出差？`

### [PENDING] 待确认态
**确认卡片模板**：
```
📋 出差申请确认

• 出差时间：{start_time} 至 {end_time}
• 出差类型：{trip_type}
• 出差事由：{reason}
• 始发地：{start_address}
• 目的地：{target_address}

确认提交请回复"确认"，取消请回复"取消"
```

### [EXECUTING] 执行态
**成功响应**：
```
✅ 出差考勤申请提交成功

申请ID：{trip_id}
出差期间视为正常出勤，请注意安全。
```

## 4. 核心约束

### 4.1 考勤规则
- 出差期间视为**正常出勤**，不计入请假或缺勤
- 不与出差购买机票/酒店、报销差旅津贴等其他流程相关

### 4.2 时间校验
- 开始时间必须早于结束时间
- 出差时段不能与已有考勤记录冲突

### 4.3 长时间出差
- 出差超过 **3 天**需附加详细的工作计划说明

### 4.4 地点要求
- 出差地点需填写具体明细，便于考勤管理人员核实

## 5. 示例对话

```
用户：我要出差
Agent：好的，请问出差时间是什么时候？
用户：明天到后天
Agent：（调用 time_oracle）请问是去国内还是国外？
用户：国内
Agent：请问出差原因是什么？
用户：拜访客户
Agent：请问从哪里出发，到哪里？
用户：北京到上海
Agent：📋 出差申请确认...

用户：确认
Agent：✅ 出差考勤申请提交成功，出差期间视为正常出勤。
```

## 6. 错误处理

| 错误场景 | 处理方式 |
|----------|----------|
| 时间冲突 | 提示"该时间段已存在考勤记录" |
| 时间逻辑错误 | 提示"结束时间必须晚于开始时间" |
| 网络错误 | 提示稍后重试 |
