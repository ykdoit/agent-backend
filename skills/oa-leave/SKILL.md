# 技能：请假申请

## 1. 业务流程

单步调用：`oa-leave-application-form`

## 2. 槽位清单

| 参数 | 说明 | 必填 | 类型 | 处理方式 |
|------|------|:----:|------|----------|
| leave_type | 请假类型 | ✅ | 枚举 | 年假/事假/病假/调休/带薪病假 |
| start_date | 开始日期 | ✅ | string | 必须调用 `time_oracle` 解析 |
| time_period | 时间段 | ✅ | 枚举 | 上午/下午 |
| leave_days | 天数 | ✅ | number | 最小单位 0.5 天 |
| reason | 原因 | ✅ | string | 用户输入 |

## 3. 状态机

### [GATHERING] 信息收集态
**任务**：逐个收集缺失参数

**追问格式（必须带序号）**：

**缺少类型时**：
```
请选择请假类型：

1. 年假
2. 事假
3. 病假
4. 调休
5. 带薪病假

请回复序号（1-5）或直接输入类型名称。
```

**缺少日期时**：
```
请问从哪天开始？（如：明天、下周一、2026-03-01）
```

**缺少时间段时**：
```
请选择时间段：

1. 上午
2. 下午

请回复序号或时间段名称。
```

**缺少天数时**：
```
请问请假几天？（最小0.5天）
```

**缺少原因时**：
```
请问请假原因是什么？
```

**用户输入映射规则**：
- 用户输入 "1" → 自动映射为第一个选项
- 用户输入 "2" → 自动映射为第二个选项
- 用户输入 "年假" → 直接使用
- 用户输入 "年" → 模糊匹配为 "年假"
- 用户输入不明确 → 追问确认

**退出条件**：所有必填参数收集完成

### [PENDING] 待确认态
**任务**：展示确认卡片，等待用户确认

**确认卡片模板**：
```
📋 请假申请确认

• 请假类型：{leave_type}
• 开始日期：{start_date}
• 时间段：{time_period}
• 请假天数：{leave_days} 天
• 请假原因：{reason}

确认提交请回复"确认"，取消请回复"取消"
```

**退出条件**：
- 用户回复"确认"/"提交"/"OK" → 进入 EXECUTING
- 用户回复"取消"/"算了" → 终止流程

### [EXECUTING] 执行态
**任务**：调用 MCP 工具提交申请

**调用参数**：
```json
{
  "leaveApplicationFormParams": {
    "leaveType": "{leave_type}",
    "leaveStartDate": "{start_date}",
    "leaveTimePeriod": "{time_period}",
    "leaveDays": "{leave_days}",
    "leaveReason": "{reason}",
    "staffDomain": "{staff_domain}"
  }
}
```

**成功响应**：
```
✅ 请假申请提交成功

申请单号：{applicationId}
请假天数：{leave_days} 天
剩余余额：{remainingBalance} 天

已进入审批流程，请注意查收审批通知。
```

**失败响应**：
```
❌ 请假申请提交失败

原因：{error_message}

请检查信息后重试，或联系人事部门。
```

## 4. 核心约束

### 4.1 序号映射规则
- Agent 必须在追问时提供**带序号的选项列表**
- 用户可回复**序号**或**选项原文**
- Agent 应智能理解用户输入，自动映射到正确选项
- 示例映射：
  - "1" → "年假"
  - "年假" → "年假"
  - "年" → "年假"（模糊匹配）

### 4.2 日期处理
- ❌ **禁止**：Agent 自行计算"明天是几号"
- ✅ **必须**：调用 `time_oracle` 解析非标准日期
- ✅ **例外**：用户已提供 YYYY-MM-DD 格式，无需调用

### 4.3 类型校验
- 只接受：`年假`、`事假`、`病假`、`调休`、`带薪病假`
- 用户说"我想休息" → 追问具体类型

### 4.4 天数计算
- 最小单位：0.5 天
- 半天请假：时间段必须是"上午"或"下午"
- 多天请假：时间段通常为空或不适用

### 4.5 证明材料提示（根据类型）
- 病假 ≥ 2 天：`提示：需要提交二级以上公立医院诊断证明`
- 其他类型：按公司政策执行

## 5. 示例对话

**示例1：完整流程（使用序号）**
```
用户：我想请假
Agent：请选择请假类型：

1. 年假
2. 事假
3. 病假
4. 调休
5. 带薪病假

请回复序号（1-5）或直接输入类型名称。

用户：1
Agent：好的，年假。请问从哪天开始？
用户：明天
Agent：（调用 time_oracle，解析为 2026-02-27）

请选择时间段：

1. 上午
2. 下午

请回复序号或时间段名称。

用户：1
Agent：好的，上午。请问请假几天？
用户：1天
Agent：请问请假原因是什么？
用户：家里有事
Agent：
📋 请假申请确认

• 请假类型：年假
• 开始日期：2026-02-27
• 时间段：上午
• 请假天数：1 天
• 请假原因：家里有事

确认提交请回复"确认"，取消请回复"取消"

用户：确认
Agent：✅ 请假申请提交成功...
```

**示例2：使用原文**
```
用户：我要请年假
Agent：好的，年假。请问从哪天开始？
用户：明天下午
Agent：请问请假几天？
用户：0.5天
Agent：请问请假原因是什么？
用户：去医院
Agent：📋 请假申请确认...
```

**示例3：用户取消**
```
用户：我想请假
Agent：请选择请假类型：
1. 年假
2. 事假
...

用户：算了，不请了
Agent：好的，已取消请假申请。有其他需要帮助的吗？
```

## 6. 错误处理

| 错误场景 | 处理方式 |
|----------|----------|
| 余额不足 | 提示剩余天数，建议调整 |
| 时间冲突 | 提示冲突日期，要求调整 |
| 权限不足 | 提示联系人事 |
| 网络错误 | 提示稍后重试 |
